# RailTwin-Q Dataset Schema Documentation

This schema documentation details the five structured, synchronized datasets generated by the RailTwin-Q simulation pipeline. These datasets serve as input layers for Layer 2 (AI Prediction) and Layer 3 (Quantum Optimization).

---

## Relational Architecture

The datasets form a relational database schema linked by `state_id`, `scenario_id`, `tick`, and entities like `train_no`, `station_id`, and `track_id`.

## 1. Train Dataset (`train_dataset.csv`)

- **Purpose**: Tracks granular train-level statuses, routes, positions, active weather/signal disruptions, route complexity features, and target delay labels.
- **Primary Key**: `scenario_id` + `tick` + `train_no`
- **Foreign Keys**:
  - `state_id` -> `network_state_dataset.csv`
  - `current_station_id` -> `station_dataset.csv`
  - `next_station_id` -> `station_dataset.csv`
  - `current_track_id` -> `track_dataset.csv`

### Column Descriptions

| Column | Data Type | Description |
| :--- | :--- | :--- |
| `scenario_id` | Integer | Unique identifier of the simulation scenario run |
| `tick` | Integer | Simulation time step in minutes |
| `state_id` | String | Unique snap identifier (`state_S{scenario:03d}_T{tick:03d}`) |
| `simulation_time_minutes` | Integer | Elasped simulation time in minutes (equals tick) |
| `time` | String | Formatted wall clock time (`HH:MM`) |
| `train_no` | Integer | Unique identifier of the train |
| `train_name` | String | Descriptive name of the train |
| `train_type` | String | Type of train (`SUPERFAST`, `EXPRESS`, `PASSENGER`, `FREIGHT`) |
| `is_priority_train` | Boolean | Binary flag (`0` or `1`) indicating priority clearance |
| `route_id` | Integer | ID of the route stopping sequence the train is following |
| `current_station_id` | Integer | ID of the station train is currently at or just left |
| `current_station` | String | Name of the station train is currently at or just left |
| `next_station_id` | Integer | ID of the next scheduled stop on the route |
| `next_station` | String | Name of the next scheduled stop on the route |
| `status` | String | Active train state (`MOVING`, `WAITING`, `ARRIVED`, `DELAYED`) |
| `progress` | Float | Percentage progress traversed on the current track (0.0 to 100.0) |
| `speed` | Float | Current speed of the train in km/h |
| `max_speed` | Float | Maximum speed allowed for this train type |
| `current_delay` | Float | Cumulative delay of this train in minutes |
| `delay_change_last_tick` | Float | Delta delay added in the last simulation tick |
| `distance_travelled` | Float | Total route distance traveled so far in km |
| `remaining_distance` | Float | Distance left on the current track segment in km |
| `remaining_route_distance` | Float | Total remaining distance to route destination in km |
| `scheduled_arrival_time` | String | Timetable scheduled arrival time (`HH:MM`) |
| `expected_arrival_time` | String | Predicted arrival time based on delay (`HH:MM`) |
| `weather_type` | String | Current weather category (`CLEAR`, `RAINY`) |
| `rain_intensity` | Float | Current rain intensity on track segment (0.0 to 1.0) |
| `signal_failure` | Boolean | Binary flag (`0` or `1`) indicating signal failure at current station |
| `maintenance` | Boolean | Binary flag (`0` or `1`) indicating maintenance block on current track |
| `track_blockage` | Boolean | Binary flag (`0` or `1`) indicating track blockage on current track |
| `festival_rush` | Boolean | Binary flag (`0` or `1`) indicating active festival scenario |
| `station_occupancy` | Float | Platform utilization percentage of current station |
| `track_occupancy` | Float | Utilization percentage of the current track segment |
| `track_speed_limit` | Float | Speed limit of the current track segment |
| `track_capacity` | Integer | Maximum train capacity of the current track segment |
| `trains_on_track` | Integer | Count of trains currently occupying this track segment |
| `remaining_stations` | Integer | Number of stations left to visit in the routestop list |
| `route_length` | Float | Total path length of the train's route in km |
| `junction_count` | Integer | Count of junctions (nodes with degree > 2) on this route |
| `alternative_routes` | Integer | Number of simple topological routes to destination |
| `critical_station_count` | Integer | Number of high-centrality nodes on this route |
| `future_delay_15` | Float | Target variable: Train delay in 15 minutes |
| `future_delay_30` | Float | Target variable: Train delay in 30 minutes |
| `future_delay_60` | Float | Target variable: Train delay in 60 minutes |
| `primary_delay_reason` | String | Main reason for delay (`SIGNAL_FAILURE`, `RAIN`, etc.) |
| `secondary_delay_reason` | String | Secondary reason for delay (`CONGESTION`, `FOLLOWING_TRAIN`, etc.) |

## 2. Station Dataset (`station_dataset.csv`)

- **Purpose**: Tracks platform occupancies, waiting queues, graph topology centralities, and future traffic congestion at stations.
- **Primary Key**: `scenario_id` + `tick` + `station_id`
- **Foreign Keys**:
  - `state_id` -> `network_state_dataset.csv`

### Column Descriptions

| Column | Data Type | Description |
| :--- | :--- | :--- |
| `scenario_id` | Integer | Unique identifier of the simulation scenario run |
| `tick` | Integer | Simulation time step in minutes |
| `state_id` | String | Unique snap identifier (`state_S{scenario:03d}_T{tick:03d}`) |
| `station_id` | Integer | Unique station identifier |
| `station_name` | String | Name of the railway station |
| `platforms_total` | Integer | Number of physical platforms available at this station |
| `platforms_available` | Integer | Platforms currently empty at this station |
| `platforms_occupied` | Integer | Platforms currently occupied by dwelling trains |
| `waiting_trains` | Integer | Count of trains queued outside waiting to enter the station |
| `incoming_trains` | Integer | Count of active trains currently moving towards this station |
| `outgoing_trains` | Integer | Count of active trains currently moving away from this station |
| `station_capacity` | Integer | Max train capacity (equivalent to platform count) |
| `station_congestion_score` | Float | Computed traffic congestion score at station (can exceed 100.0) |
| `station_utilization_percent` | Float | Percentage of platform occupancy (0.0 to 100.0) |
| `average_station_delay` | Float | Average delay in minutes of trains currently at the station |
| `weather_type` | String | Current weather category (`CLEAR`, `RAINY`) |
| `active_disruptions` | String | Active station level event tags (e.g. `SIGNAL_FAILURE`, `POWER_FAILURE`) |
| `node_degree` | Integer | NetworkX node degree (number of connected track links) |
| `betweenness_centrality` | Float | NetworkX betweenness centrality score of this station |
| `closeness_centrality` | Float | NetworkX closeness centrality score of this station |
| `station_connectivity` | Float | Topological connectivity metric (average neighbor degree) |
| `future_station_congestion` | Float | Target variable: Station congestion score in 30 minutes |

## 3. Track Dataset (`track_dataset.csv`)

- **Purpose**: Tracks track utilization, capacities, average speeds, scheduled maintenance blocks, and future congestion targets.
- **Primary Key**: `scenario_id` + `tick` + `track_id`
- **Foreign Keys**:
  - `state_id` -> `network_state_dataset.csv`
  - `source_station_id` -> `station_dataset.csv`
  - `destination_station_id` -> `station_dataset.csv`

### Column Descriptions

| Column | Data Type | Description |
| :--- | :--- | :--- |
| `scenario_id` | Integer | Unique identifier of the simulation scenario run |
| `tick` | Integer | Simulation time step in minutes |
| `state_id` | String | Unique snap identifier (`state_S{scenario:03d}_T{tick:03d}`) |
| `track_id` | Integer | Unique track segment identifier |
| `source_station_id` | Integer | ID of the source station |
| `source_station` | String | Name of the source station |
| `destination_station_id` | Integer | ID of the destination station |
| `destination_station` | String | Name of the destination station |
| `distance` | Float | Length of the track segment in km |
| `track_capacity` | Integer | Maximum train capacity allowed on this track segment |
| `trains_on_track` | Integer | Count of trains currently occupying the track segment |
| `occupancy_percent` | Float | Percentage capacity utilized (0.0 to 100.0) |
| `average_speed` | Float | Average speed in km/h of trains traversing this track |
| `travel_time` | Float | Estimated traversal time in minutes under current conditions |
| `blocked` | Boolean | Binary flag (`0` or `1`) indicating track blockage disruption |
| `maintenance` | Boolean | Binary flag (`0` or `1`) indicating maintenance closure |
| `future_track_congestion` | Float | Target variable: Track occupancy percent in 30 minutes |

## 4. Network State Dataset (`network_state_dataset.csv`)

- **Purpose**: Chronological network-wide state history mapping global congestions, utilizations, active disruptions, and resilience metrics. Exposes inputs for Layer 3 (Quantum Scheduler).
- **Primary Key**: `scenario_id` + `tick` + `state_id`

### Column Descriptions

| Column | Data Type | Description |
| :--- | :--- | :--- |
| `scenario_id` | Integer | Unique identifier of the simulation scenario run |
| `tick` | Integer | Simulation time step in minutes |
| `state_id` | String | Unique snap identifier (`state_S{scenario:03d}_T{tick:03d}`) |
| `simulation_time_minutes` | Integer | Elapsed simulation time in minutes (equals tick) |
| `time_of_day` | String | Time category (`MORNING`, `AFTERNOON`, `EVENING`, `NIGHT`) |
| `day_type` | String | Day category (`WEEKDAY`, `WEEKEND`, `FESTIVAL`) |
| `network_congestion_score` | Float | Global weighted congestion score (0.0 to 100.0) |
| `average_network_delay` | Float | Average delay of all trains in the network in minutes |
| `platform_utilization_percent` | Float | Percentage of all station platforms currently in use |
| `track_utilization_percent` | Float | Percentage of all track capacities currently in use |
| `active_disruptions` | String | Comma-separated list of active event names (e.g. `HEAVY_RAIN`, `SIGNAL_FAILURE`) |
| `num_active_trains` | Integer | Count of trains currently active in the network |
| `congested_stations` | Integer | Count of stations with platforms occupied >= 80% |
| `congested_tracks` | Integer | Count of tracks with occupancy percent >= 80% |
| `average_train_speed` | Float | Average speed of all trains in the network in km/h |
| `total_waiting_trains` | Integer | Total trains waiting to enter platforms across the network |
| `total_platforms_used` | Integer | Total count of platforms occupied across the network |
| `total_tracks_used` | Integer | Total count of tracks occupied by at least one train |
| `network_resilience_score` | Float | Performance index indicating system recovery cushion (0.0 to 100.0) |
| `recovery_time_estimate` | Float | Estimated minutes until network-wide delays resolve to zero |
| `bottleneck_count` | Integer | Sum of congested stations and tracks (>80% utilization) |
| `critical_track_count` | Integer | Count of tracks undergoing maintenance or blockages |
| `critical_station_count` | Integer | Count of stations suffering from signal or power failures |
| `future_network_congestion` | Float | Target variable: Network congestion score in 30 minutes |
| `future_platform_utilization` | Float | Target variable: Global platform utilization in 30 minutes |
| `future_average_network_delay` | Float | Target variable: Average network delay in 30 minutes |

## 5. Delay Propagation Dataset (`delay_propagation_dataset.csv`)

- **Purpose**: Captures explicit directed cascade links between delayed trains to model cascade delay propagation paths.
- **Primary Key**: `scenario_id` + `tick` + `parent_train` + `child_train`
- **Foreign Keys**:
  - `state_id` -> `network_state_dataset.csv`
  - `parent_train` -> `train_dataset.csv` (`train_no`)
  - `child_train` -> `train_dataset.csv` (`train_no`)
  - `source_station_id` -> `station_dataset.csv`
  - `affected_station_id` -> `station_dataset.csv`
  - `intermediate_station_id` -> `station_dataset.csv`

### Column Descriptions

| Column | Data Type | Description |
| :--- | :--- | :--- |
| `scenario_id` | Integer | Unique identifier of the simulation scenario run |
| `tick` | Integer | Simulation time step in minutes |
| `state_id` | String | Unique snap identifier (`state_S{scenario:03d}_T{tick:03d}`) |
| `parent_train` | Integer | Train number of the train causing the delay |
| `child_train` | Integer | Train number of the train experiencing the cascading delay |
| `source_station_id` | Integer | ID of the station where the parent train is located |
| `source_station` | String | Name of the station where the parent train is located |
| `affected_station_id` | Integer | ID of the station where the child train is located |
| `affected_station` | String | Name of the station where the child train is located |
| `intermediate_station_id` | Integer | ID of the station where the conflict occurred (N/A if track-based) |
| `intermediate_station` | String | Name of the station where the conflict occurred (N/A if track-based) |
| `delay_transferred` | Float | Delay minutes added to the child train in this simulation tick |
| `propagation_depth` | Integer | Distance of this node from the primary source delay node in the DAG |
| `delay_chain_length` | Integer | Total length of the cascade propagation sequence |
| `cause` | String | Logic event that triggered delay transfer (`WAITING_FOR_PLATFORM`, `FOLLOWING_TRAIN`) |
| `network_congestion_score` | Float | Global network congestion score at this tick |
| `future_delay` | Float | Target variable: Delay of the child train in 30 minutes |
